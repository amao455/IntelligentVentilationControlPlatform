# Codex 提示词：智能通风管控平台前台展示端全界面生成说明（极简可执行版）

## 1. 任务目标
请直接生成一个**完整可运行的前端项目**，项目名称为“智能通风管控平台前台展示端”。

目标不是做宣传官网，而是做一个**工业软件风格**的矿井智能通风业务平台，界面要求：

- 脱离 AI 海报风、科幻宣传风、玻璃拟态风；
- 风格克制、稳重、专业，符合工业软件、中后台平台、调度监控系统审美；
- 信息密度中高，布局规整，强调状态、结构、层级和可读性；
- 适合桌面端 1920×1080 使用，同时兼容 1440、1366 宽度；
- 所有页面均使用同一套布局系统、配色系统、卡片系统和状态语义；
- 页面要看起来像真实可交付项目，而不是概念图。

请输出：

1. 完整项目代码；
2. 完整路由；
3. 完整布局组件；
4. 所有页面的静态高保真界面；
5. 页面中的示例数据、图表、表格、状态块、流程块、卡片等；
6. 代码结构清晰，可继续二次开发。

---

## 2. 技术要求
前端技术栈默认采用：

- React
- TypeScript
- Vite
- Ant Design
- ECharts
- React Router

如你认为更适合，也可使用：

- Zustand 进行状态管理；
- dayjs 处理时间；
- CSS Modules 或 less；
- 图标使用 Ant Design Icons。

不要使用过度炫技的动画库，不要引入对工业软件无必要的复杂依赖。
